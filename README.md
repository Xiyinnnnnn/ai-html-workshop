## 1. 群策.html — Multi-Agent 群聊协作平台

### 功能概述
一个模拟群聊的 AI 协作环境，多个具有不同专业技能的 AI Agent 在同一个群中讨论、提议、表决。用户可作为群主参与讨论或下达指令。

### 内置 Agent（6 个）
- **Strategos（拆解者）** — 复杂问题拆解与规划
- **Archonis（管理架构师）** — 组织结构与流程设计
- **Cortext（代码架构师）** — 软件架构与技术方案
- **Lyra（叙事者）** — 故事架构与文学创作
- **Axion（机制设计师）** — 游戏机制与数值设计
- **Vex（策略顾问）** — 销售策略与客户分析

### 技术亮点
- **JSON 命令块协议**：Agent 间通过结构化 JSON 通信，自动解析和过滤
- **提议/表决状态机**：Agent 发起提案 → 其他成员投票 → 法定人数判定 → 5 分钟超时自动关闭
- **并发控制 + 唤醒调度器**：可设定 Agent 唤醒间隔，避免 API 刷屏
- **上下文拼接引擎**：自动注入历史摘要 + 成员列表，每个 Agent 知晓群内全貌
- **角色系统**：群主(Host) / 管理员(Admin) / 普通成员，支持禁言/拉人/删人
- **消息折叠**：长文本自动折叠 + 思考过程可展开
- **响应式布局**：桌面端侧边栏 + 移动端底部 Tab 切换

### 使用方式
1. 设置 API Key
2. 创建群聊，拉入 Agent
3. 输入消息开始讨论，或使用 `@Agent名` 指定对话对象
4. Agent 会自动发起提议并相互表决

## 2. 游戏.html — 文字冒险游戏引擎 5.0

### 功能概述
一个拥有 108 个 AI Agent 的文字冒险游戏引擎，覆盖从世界创世到微观战斗的全维度叙事。L0 主控 Agent 统一调度，根据场景智能分派子 Agent。

### Agent 五层架构

| 层级 | 数量 | 职责 | 示例 Agent |
|------|------|------|-----------|
| **L0** | 1 | 主控决策 | 接收全量状态 + 玩家输入，签发命令 |
| **L1** | 11 | 领域总管 | 对话/战斗/探索/氛围/经营/战略/推演/创世/亲密/暴力/政治 |
| **L2** | 25 | 功能执行 | 对话编排/NPC扮演/战斗回合/场景呈现/恐怖氛围/战报/外交/时间推进/叙事线/节奏控制等 |
| **L3** | 54 | 原子操作 | 开场/推进/转折/攻击判定/伤口生成/感官四觉/成人链/血腥链/政治链等 |
| **L4** | 12 | 数据查询 | 纯数据检索，不做创造性描述 |
| **L5** | 5 | 工具 | 骰子/名称生成/语法检查/翻译/摘要 |

### 命令协议（L0 → 子 Agent）
- `QUERY` — 查询世界状态
- `RESPOND` — 快餐回合直接响应
- `DISPATCH` — 调度指定子 Agent
- `REVISE` — 修改子 Agent 输出
- `FINALIZE` — 整合签发最终输出
- `SYSTEM` — 系统操作（存档/导入/新游戏）

### 技术亮点
- **创世流程**：种子确认 → 细化需求 → 框架展示 → 用户确认 → 落地，四阶段多轮对话
- **世界状态系统**：HP/SAN 状态条、游戏时间追踪、铜币经济、物品栏
- **打字机效果**：文本逐字输出，沉浸感强
- **存档系统**：导入/导出/预览 + 超时弹窗处理
- **快捷键**：[1-4] 快速选择 / [Enter] 发送 / [Esc] 取消 / [F1] 调试 / [Ctrl+S] 设置

### 使用方式
1. 配置 API Key
2. 点击「新游戏」，输入世界种子（如「赛博朋克」「修仙世界」）
3. 跟随 Agent 的多轮引导完成世界创建
4. 通过选项按钮或自定义输入推进故事

## 3. 小说.html — NovelCraft vFINAL+

### 功能概述
AI 驱动的小说创作全流程系统，从需求解析到终章完成，21 个专业子 Agent 在主编调度下分工协作。

### 21 个子 Agent 一览

| ID | Agent | 职责 |
|----|-------|------|
| A0 | 需求解析 | 从创作种子提取结构化需求 |
| A1 | 世界观构建 | 地理/历史/体系/势力/不可变规则 |
| A2 | 角色设计 | 性格/外貌/弧光/关系网 |
| A3 | 大纲生成 | 章节节点/冲突曲线/伏笔 |
| A4 | 章节写作 | 实际内容生成 |
| A5 | 风格润色 | 文风统一 |
| A6 | 摘要切片 | 长文本压缩 |
| A7 | 记忆检索 | 历史信息提取 |
| A8 | 事实核查 | 前后一致性验证 |
| A9 | 冲突调解 | 设定矛盾解决 |
| A10 | 评审反馈 | 10 维评分体系 |
| A11 | 任务书撰写 | 为子 Agent 准备指令 |
| A12 | 状态同步 | 角色/物品/伏笔自动更新 |
| A13 | 评审决策顾问 | pass/revise/reject 建议 |
| A14 | 阶段规划顾问 | 流程推进建议 |
| A15 | 大纲扩展 | 增量生成新节点 |
| A16 | 角色检测 | 从章节中识别新角色 |
| A17 | 剧情诊断 | 全篇节奏/弧光健康检查 |
| A18 | 消息润色 | 主编→用户沟通优化 |
| A19 | 上下文组装 | 写作上下文包准备 |
| A20 | 项目审计 | 全局健康度评估 |

### 主编 8 种命令
`ask_user` / `present_to_user` / `call_subagent` / `evaluate_result` / `move_phase` / `request_user_decision` / `complete` / `update_state`

### 九步工作流
种子 → 需求对齐 → 世界观 → 角色 → 大纲 → 大纲审阅 → 写作 → 评审 → 完成

### 技术亮点
- **Dexie.js 持久化**：多项目管理 + IndexedDB 存储
- **10 维评分体系** + 自动修订（最多 3 次）
- **里程碑检测**：10/25/50/100/150/200 章自动触发诊断或审计
- **检查点/回滚**：支持存点 + 恢复
- **TXT/JSON 导入导出**：完整项目数据可迁移

### 使用方式
1. 配置 API Key
2. 新建项目，输入创作种子（一句话描述即可）
3. 主编自动推进工作流，关键节点会询问你的意见
4. 左侧边栏可随时查看世界观/角色/大纲/章节

## 4. 色色.html — 色色驱动 Pro

### 功能概述
成人向互动叙事引擎，支持角色卡管理、叙事模块训练、大纲自动生成、偏好精细设置。

### 核心功能
- **角色卡系统**：AI 生成 / 手动编辑 / 导入导出
- **叙事模块训练**：从范文学习文风、词汇库 → 保存为档位随时切换
- **大纲生成**：128k tokens 预算超详细色情大纲
- **偏好设置**：故事方式/长度/场景/玩法/体位/限制/高潮要求/射精部位
- **选项自动生成**：每轮基于当前剧情自动提供 3-6 个后续选项
- **历史管理**：自动摘要压缩 + token 估算

### 技术亮点
- **可训练叙事模块**：JSON 结构的人格配置（性格/风格/词汇库/禁忌词）
- **自动重试**：API 调用失败 3 次自动重试
- **请求取消**：支持 AbortController 手动取消
- **多模型管理**：可添加/切换不同模型

## 5. 脏话.html — 脏话生成 v6.0

### 功能概述
AI 驱动的攻击性文本生成器，内置 369 条精标语录库，支持 7 种角色风格、5 个场景预设、20+ 种原子技能自动组合。

### 生成流水线
1. **成分分析（L1）**：AI 分析目标原文，自动识别 20+ 物种标签（小仙女/龟男/键盘侠/杠精/原批/果蛆等）
2. **技能装载（L2）**：根据攻击强度 + 角色 + 场景，自动装配技能组合
3. **战术编排（L3-L5）**：AI 基于装载的技能生成攻击文本
4. **和谐处理（L6）**：低强度时自动将敏感词替换为谐音/拼音缩写

### 可选角色风格
战斗型喷子 / 反串型喷子 / 总结型喷子 / 阴阳大师 / 歇后语大师 / 对联狂魔 / 混乱邪恶型

### 场景预设
下班红温型 / 深夜破防型 / 游戏被坑型 / 饿肚暴躁型 / 正义暴怒型

### 技术亮点
- **固定强度技能表**：10 级强度对应不同技能组合
- **语录库 AI 精标**：四维标签（强度/风格/目标/情绪）自动标注
- **语录库管理**：导入/导出/重置/手动增减
- **历史记录**：所有生成结果可回溯 + 一键复制
